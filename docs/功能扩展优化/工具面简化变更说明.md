## 新增内部方法

### `handleIndex` (handlers.ts:348-395)
- 调用 `handleIndexCodebase` → 向量索引
- 调用 `handleIndexRepository` → 图索引
- 已索引项目自动跳过（`stats.nodes > 0`），`force=true` 时重建
- 图索引失败时返回警告，不影响向量索引结果
- 合并图索引结果到响应

### `handleStatus` (handlers.ts:1116-1150)
- 合并 `handleGetIndexingStatus`（向量） + `handleIndexStatus`（图）
- 图状态查询失败时降级返回错误信息

### `handleSearchCode` 图增强 (handlers.ts:951-980)
- 向量搜索后自动查询图索引
- 使用 `exactFilePath` 精确匹配（避免 `%path%` 子串误匹配）
- 取前 3 个结果文件的图节点，附加到搜索结果末尾

---

## 已知风险点（建议排查）

1. **`handleIndex` 图索引失败处理**: 图索引失败时返回警告但不阻断，确保向量索引不受影响
2. **`handleSearchCode` 图增强性能**: 每次搜索额外查询 3 次图索引，大项目可能增加延迟
3. **`exactFilePath` 匹配**: 新增字段，确认 `findNodes` 的 countQuery 也正确处理（当前仅主查询有）
4. **向后兼容**: 旧工具名（`index_codebase` 等）作为别名保留，确认无调用方使用已移除的图工具名
5. **`graphToolHandlers` 为 null**: 若图引擎未初始化，`handleIndex`/`handleSearchCode`/`handleStatus` 应优雅降级
