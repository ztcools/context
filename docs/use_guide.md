# Claude Context MCP 使用教程

## 前置条件

MCP 已配置好，Attu 后端已部署，此处不涉及部署步骤。

---

## 四个工具

配置完成后，IDE 会识别到 4 个 MCP 工具（兼容旧名称）：

| 工具 | 旧名称 | 用途 |
|------|--------|------|
| `index` | `index_codebase` | 为项目建立语义索引 + 知识图谱，一次调用完成双索引 |
| `search` | `search_code` | 语义搜索 + 图上下文增强（调用链、架构归属、死代码检测） |
| `status` | `get_indexing_status` | 查看索引状态 + 知识图谱架构概览 |
| `clear` | `clear_index` | 清除向量索引 + 图索引 |

---

## 实际使用

### 1. 索引项目

在对话中直接说：

> "索引当前项目"
> "index this project"
> "帮我把 /path/to/project 索引一下"

Agent 会自动调用 `index`。同一仓库(url+branch)只需索引一次，团队成员共享索引。

**首次索引**: 全量构建向量索引 + 知识图谱（节点、调用关系、路由等）。  
**后续索引**: 自动检测 git 变更文件，仅增量更新图索引，无需全量重建。  
**强制重建**: 使用 `force=true` 参数全量重建。

### 2. 搜索代码

在对话中直接说：

> "这个项目的认证逻辑是怎么实现的？"
> "帮我找到所有数据库连接相关的代码"
> "这个 bug 可能出在哪里？"

Agent 会自动调用 `search`，返回结果包含三层信息：

| 层级 | 内容 | 示例 |
|------|------|------|
| 语义搜索 | 匹配的代码片段 | `authenticate()` 函数实现 |
| 图上下文 | 调用者/被调用者/死代码标记 | `authenticate ← login(), refreshToken()` |
| 调用链 | 深度调用链追踪 | `login → authenticate → validateToken → hasRole` |
| 架构 | 所属模块和入口点 | `Module: src/auth (cohesion: 0.85)` |

**向量搜索无结果时自动 fallback 到图搜索**，确保总能找到相关代码。

### 3. 查看索引状态

> "当前项目索引状态是什么？"
> "看看索引进度"

Agent 调用 `status`，返回：
- 向量索引状态（Milvus 进度、chunk 数）
- 图索引状态（节点数、边数、类型分布、路由数）
- 架构概览（入口点、模块聚类、内聚度）

### 4. 清除索引

> "清除当前项目的索引"
> "重新索引这个项目"

Agent 调用 `clear` 或 `index`（带 force=true）。

## 关键行为

- **不传路径默认为当前工作区**：对话中直接说"索引当前项目"即可，无需指定路径
- **url+branch 隔离**：同一仓库不同 clone 路径共享索引，不重复建立
- **团队共享**：A 成员索引后，B 成员自动发现已索引，无需重复
- **智能增量索引**：后续索引自动检测 git 变更，仅索引变更文件
- **搜索自动降级**：未索引的项目使用 `search` 会报错，提示先索引
- **图增强透明**：图操作失败不影响向量搜索，自动降级
- **双工具名兼容**：新旧工具名均可使用（`index`/`index_codebase` 等）

## 环境变量（可选）

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `CLAUDE_CONTEXT_BACKGROUND_SYNC` | 是否启用后台自动同步 | `true` |
| `CLAUDE_CONTEXT_SYNC_INTERVAL_MS` | 后台同步间隔(ms) | `300000` (5分钟) |
| `CLAUDE_CONTEXT_TRIGGER_WATCHER` | 是否启用即时触发同步 | `true` |
