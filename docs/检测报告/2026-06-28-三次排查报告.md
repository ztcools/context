## 二、新发现问题

### P1 — 影响功能正确性

| # | 文件 | 问题 |
|---|------|------|
| 1 | `milvus-restful-vectordb.ts:827` | `checkCollectionLimit()` 是 TODO 占位，永远返回 `true`。REST API 路径下无法检测 collection 数量上限，超过限制时错误信息不友好。 |
| 2 | `graph-store.ts:239-258` | `findNodes` 的 degree 过滤器 (minDegree/maxDegree) 在 SQL 查询后应用，但 `total` count 查询未包含这些过滤条件。当使用 degree 过滤时，返回的 `total` 和 `hasMore` 不准确。 |

### P2 — 代码质量

| # | 文件 | 问题 |
|---|------|------|
| 3 | `context.ts:1103` | `getLanguageFromExtension` 缺少 `.md`/`.markdown` 映射，但两者在 `DEFAULT_SUPPORTED_EXTENSIONS` 中。虽不影响功能（会 fallback 到 text），但映射不完整。 |
| 4 | `context.ts:1250` | `require('os').homedir()` 在 ESM 模块中用 CJS require，且 `os` 未在文件顶部 import。 |
| 5 | `context.ts:1474,1496,1512` | `getSplitterInfo` / `isLanguageSupported` / `getSplitterStrategyForLanguage` 三个方法每次调用都 `require('./splitter/ast-splitter')`，应缓存引用。 |
