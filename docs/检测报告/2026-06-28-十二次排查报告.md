## 二、新发现问题

### P2 — 代码质量

| # | 类型 | 文件 | 问题 |
|---|------|------|------|
| 1 | **新发现** | `gemini-embedding.ts:75-104` `voyageai-embedding.ts:73-98` `ollama-embedding.ts:91-` | **其他 embedding provider 无重试逻辑**：OpenAI 已添加 429/5xx 重试，但 Gemini、VoyageAI、Ollama 的 `embedBatch` 均无重试。遇到限流或临时故障时直接丢弃 chunk buffer。应统一各 provider 重试策略。 |
| 2 | **新发现** | `searcher.ts:170` → `utils.ts:7` → `graph-store.ts:578` | **`regexToLike` 未转义 LIKE 通配符**：`escapeRegex` 转义了正则特殊字符，但 `regexToLike` 在剥离转义符后，`%` 和 `_` 是 LIKE 通配符。若文件路径含 `%`（如 `100%_test.js`），LIKE 匹配会错误匹配其他文件。 |

### P3 — 边界情况

| # | 类型 | 文件 | 问题 |
|---|------|------|------|
| 3 | **新发现** | `context.ts:790-798` | **`prepareCollection` 中 drop 先于 dimension 检测**：`forceReindex` 时先 drop 旧 collection，再 `detectDimension()`。若 dimension 检测失败（API key 无效、网络错误），旧 collection 已删除且新 collection 未创建，codebase 丢失索引。应将 drop 延后至 dimension 检测成功后。 |
| 4 | **新发现** | `graph-handlers.ts:426-435` | **`handleDetectChanges` git diff 语义不一致**（前次未修复）：`git diff --name-only ${baseBranch}...HEAD` 失败时 fallback 到 `git diff HEAD`。前者显示 baseBranch 以来所有变更，后者仅显示未提交变更。fallback 结果与预期语义完全不同。 |
| 5 | **新发现** | `sync.ts:160-161` | **`getIndexedCodebases()` 返回 identities 但错误日志打印了 identities**：`console.log` 打印 `indexedCodebases` 数组内容为 identities 而非 localPath，调试时可读性差。 |

## 四、结论

前十一轮 79 个问题已修复（P3 除外）。第十二轮发现 5 个新问题，均为新发现。

**重点关注：**
- **P2-1 新发现**：Gemini、VoyageAI、Ollama 的 `embedBatch` 均无重试逻辑。OpenAI 已修复，应统一到其他 provider。
- **P2-2 新发现**：`regexToLike` 剥离转义符后未转义 `%` 和 `_` 通配符。文件路径含 `%` 时 LIKE 匹配范围过大。修复：在 `regexToLike` 中对 `%` 和 `_` 添加 `\` 转义。
- **P3-3 新发现**：`prepareCollection` 中 drop 先于 dimension 检测，检测失败时旧 collection 已丢失。应将 drop 延后。

累计 84 个问题，0 个 P0。P2 问题呈下降趋势，P3 问题为防御性优化。
