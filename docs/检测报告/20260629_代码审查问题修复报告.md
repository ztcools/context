## 三、疑似 Bug

### 3.1 `graph-handlers.ts` idMap 映射策略有隐患

[packages/mcp/src/graph-handlers.ts:150-157](file:///home/zt/claude-context/packages/mcp/src/graph-handlers.ts#L150-L157)

```typescript
const idMap = new Map<number, number>();
for (const node of result.nodes) {
    const realId = graphBuffer.upsertNode(...);
    idMap.set(idMap.size, realId);  // 假设 extractor 返回的 edge.sourceId 从 0 开始自增
}
for (const edge of result.edges) {
    const sourceId = idMap.get(edge.sourceId);  // 依赖 extractor 内部 ID 分配顺序
    ...
}
```

`idMap` 用 `idMap.size` 作为 key，假设 `extractor.extract()` 返回的 `edge.sourceId` 是 0-based 自增的。如果 extractor 内部 ID 不是从 0 开始（比如并发场景），边映射会失败。建议改为 `node.id` 或 `node.qualifiedName` 作为 key。

### 3.2 `synchronizer.ts` 空 extensions 时 git ls-files 参数异常

[packages/core/src/sync/synchronizer.ts:58-62](file:///home/zt/claude-context/packages/core/src/sync/synchronizer.ts#L58-L62)

```typescript
const extPatterns = this.supportedExtensions.map((e) => `"*${e}"`).join(' ');
const output = execSync(
    `git -C "${dir}" ls-files --cached --others --exclude-standard -- ${extPatterns}`,
```

当 `supportedExtensions` 为空数组时，`extPatterns` 为 `""`，`git ls-files --cached --others --exclude-standard -- ` 会列出**所有** git 文件（包括图片、二进制等），导致后续 `hashFile` 阶段读取大量无用文件。应在 `supportedExtensions.length === 0` 时跳过或直接返回空。
