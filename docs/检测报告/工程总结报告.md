## 三、集成 codebase-memory-mcp 的功能

采用**按需集成**策略，仅引入 claude-context 没有的部分，保留原有算法与架构：

| codebase-memory-mcp 功能 | 集成方式 | 文件 |
|--------------------------|----------|------|
| InMemoryGraphBuffer | TS 重构核心逻辑 | `packages/graph/src/graph-buffer.ts` |
| FunctionRegistry（4 步调用解析） | TS 重构，含 import_map / same_module / unique_name / suffix_match 策略 | `packages/graph/src/registry.ts` |
| 架构分析（ArchitectureAnalyzer） | TS 重构，含聚类、入口点检测、内聚度 | `packages/graph/src/architecture.ts` |
| 调用链追踪（GraphTracer） | TS 重构 | `packages/graph/src/tracer.ts` |
| 图上下文增强搜索 | 集成到原有 search 流程 | `packages/mcp/src/handlers.ts` |
| 增量索引 + 变更检测 | 集成到原有 index 流程 | `packages/mcp/src/graph-handlers.ts` |

**未集成部分**：codebase-memory-mcp 的向量索引（claude-context 已有 Milvus 向量索引，保留原逻辑）。

**关键差异**：codebase-memory-mcp 是纯图索引（先构图再向量化图），claude-context 是代码片段向量化 + 图索引双轨并行，两者互补。
