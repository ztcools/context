## 二、新发现问题

### P2 — 代码质量

| # | 类型 | 文件 | 问题 |
|---|------|------|------|
| 1 | **连锁反应** | `graph-handlers.ts:94-99` | **`deleteProject` 在事务外执行**：`mode='full'` 时 `deleteProject(project)` 在 `beginTransaction()` 之前调用。若后续文件解析失败触发 `rollbackTransaction`，事务回滚但 `deleteProject` 已提交（自动提交），旧数据不可恢复。应将 `deleteProject` 移入事务内。 |
| 2 | **新发现** | `handlers.ts:440-453,498-505` | **`forceReindex` 两次 `removeCodebaseCompletely`**：若 codebase 正在索引且 forceReindex=true，line 440-453 先处理旧任务并移除 snapshot，line 498-505 再次执行 `removeCodebaseCompletely` + `clearIndex`。第二次调用虽幂等，但 `saveCodebaseSnapshot` 被调用两次，产生不必要的 I/O。 |

### P3 — 边界情况

| # | 类型 | 文件 | 问题 |
|---|------|------|------|
| 3 | **新发现** | `graph-handlers.ts:426-435` | **`handleDetectChanges` git diff 语义不一致**：`git diff --name-only ${baseBranch}...HEAD` 失败时 fallback 到 `git diff HEAD`。前者显示「baseBranch 以来所有变更」，后者仅显示「未提交变更」。若仓库有未 push 的本地提交，fallback 结果完全错误，且用户无感知。 |
| 4 | **新发现** | `graph-handlers.ts:849` | **`scanFiles` 忽略列表不完整**：缺少 `.venv`、`vendor`、`target`、`coverage`、`.cache`、`.idea`、`.vscode`、`.circleci`、`bin`、`obj`、`out`、`tmp`、`temp`、`.tox`、`.mypy_cache`、`.pytest_cache`、`.turbo` 等常见目录。 |
| 5 | **新发现** | `context.ts:517-520` | **`deleteFileChunks` query+delete 非原子操作**：先 query 获取 ID 列表，再 delete。若在两次调用之间 collection 被清除或 ID 被修改，delete 会静默失败。Milvus 支持 `delete by filter` 可直接删除。 |
| 6 | **新发现** | `graph-handlers.ts:143-145` | **`resolveCrossFileCalls` 在事务内但无事务保护**：`resolveCrossFileCalls` 在 `commitTransaction` 之前调用，其内部 `upsertEdge` 操作在事务内。但如果 `resolveCrossFileCalls` 失败（如 N+1 查询超时），整个事务回滚，包括已处理的文件节点。跨文件调用解析失败会导致整个索引失败而非仅跳过跨文件调用。 |
| 7 | **新发现** | `handlers.ts:440-453` | **`forceReindex` 取消旧任务后不验证旧任务已终止**：`await oldTask.promise` 后直接 `delete`，但 `IndexAbortError` 不会在 `startBackgroundIndexing` 中抛出（被 catch 吞掉且 return）。`await` 只会等到 `finally` 执行完毕，但旧任务可能仍在 `processChunkBuffer` 中。 |

## 四、结论

前九轮 66 个问题已修复（P3 除外）。第十轮发现 7 个新问题，其中 1 个为连锁反应，6 个为新发现。

**重点关注：**
- **P2-1 连锁反应**：`deleteProject` 在事务外执行，`mode='full'` 索引失败时旧数据不可恢复。应将 `deleteProject(project)` 移入 `beginTransaction` 之后。
- **P2-2 新发现**：`forceReindex` 的 `removeCodebaseCompletely` + `saveCodebaseSnapshot` 被调用两次，第二次是冗余操作。
- **P3-6 新发现**：`resolveCrossFileCalls` 失败会导致整个事务回滚，包括已成功处理的文件节点。跨文件调用解析失败应该降级为警告而非回滚整个索引。

累计 73 个问题，0 个 P0。代码质量持续提升，核心功能稳定。P3 问题为防御性优化，建议逐步修复。
