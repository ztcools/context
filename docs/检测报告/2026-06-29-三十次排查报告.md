## 二、新发现

### P3 — Merkle DAG `addNode` 重复 rootIds

**文件**: `packages/core/src/sync/merkle.ts`  
**行号**: 38-40  
**严重程度**: P3  

```typescript
public addNode(data: string, parentId?: string): string {
    const nodeId = this.hash(data);
    // ...
    if (parentId) {
        // ...
    } else {
        this.rootIds.push(nodeId);  // 同一数据无 parent 时重复 push
    }
    this.nodes.set(nodeId, node);  // 覆盖已有节点
    return nodeId;
}
```

**问题**: 调用 `addNode(sameData)` 两次（无 parent）时，`rootIds` 数组会包含**重复的 ID**，导致 `getRootNodes()` 返回重复节点。虽然当前 `synchronizer.ts` 中 `buildMerkleDAG` 每次创建新实例不会触发此问题，但 `MerkleDAG` 类本身存在潜在 bug。

**修复建议**: 在 `rootIds.push(nodeId)` 前检查 `this.nodes.has(nodeId)`，已存在则跳过。
