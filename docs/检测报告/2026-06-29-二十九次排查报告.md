## 二、新发现

### P1 — RESTful Milvus 认证类型错误

**文件**: `packages/core/src/vectordb/milvus-restful-vectordb.ts`  
**行号**: 156  
**严重程度**: P1  

```typescript
// 错误写法
headers['Authorization'] = `Bearer ${this.config.username}:${this.config.password}`;
```

**问题**: `Bearer` 认证要求传入单个 token 字符串，而此处传入了 `username:password` 格式。正确的认证方式应为 `Basic` 认证：

```typescript
// 正确写法
const credentials = Buffer.from(`${this.config.username}:${this.config.password}`).toString('base64');
headers['Authorization'] = `Basic ${credentials}`;
```

**影响**: 使用用户名/密码认证的 RESTful Milvus 连接会**始终失败**，因为服务端收到的 `Bearer myuser:mypass` 不是有效的 token 格式。

### P2 — `calculateCohesion` 冗余查询

**文件**: `packages/graph/src/architecture.ts`  
**行号**: 167, 184  
**严重程度**: P2  

`clusterByDirectory` 已通过 `getEdgesBySourceBatch(nodeIds)` 批量加载了所有节点的边（L152-153），但随后调用 `calculateCohesion(dirNodes)` 时，`calculateCohesion` 内部又逐节点调用 `getEdgesBySource(node.id)` 重新查询相同的边。

**修复建议**: 将 `edgesBatch` 作为参数传入 `calculateCohesion`，避免重复查询。
