## 四、发现的 1 个问题

### ⚠️ P3 — 增量索引失败时提示信息不准确

**位置**: `handlers.ts` L403-L413  
**严重程度**: P3（不影响功能，不影响上线）

**问题描述**: 当 `detectChanges` 成功检测到变更文件，但后续 `handleIndexRepository` 增量索引失败时，用户看到的是"no changes detected, use force=true to re-index"，而实际是增量索引执行失败。

**复现条件**: `detectChanges` 返回非空变更文件列表，但 `handleIndexRepository` 的 `files` 参数传入后索引出错。

**修复方案**:
```typescript
// 当前代码 L403-L413
} catch (e: any) {
    console.warn(`[INDEX] Incremental graph indexing failed: ${e.message}`);
}
// 之后走到 "No changes detected" 分支

// 建议改为：
} catch (e: any) {
    console.warn(`[INDEX] Incremental graph indexing failed: ${e.message}`);
    const vectorText = vectorResult.content[0]?.text || '';
    return {
        ...vectorResult,
        content: [{ type: 'text', text: vectorText +
            `\n\n⚠️ Graph incremental indexing failed: ${e.message}` +
            `\n  Use force=true to re-index fully.` }],
    };
}
```

**影响评估**: 极低。仅在 `detectChanges` 成功但 `handleIndexRepository` 失败时触发，概率很低。
