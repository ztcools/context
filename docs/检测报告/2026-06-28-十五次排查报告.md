### P3 — 边界情况

| # | 类型 | 文件 | 问题 |
|---|------|------|------|
| 2 | **新发现** | `graph-handlers.ts:68-133` | **图索引事务粒度过大**：整个 `for` 循环在一个事务内，若第 N 个文件 `extract()` 失败，前 N-1 个文件的 `upsertNode`/`upsertEdge` 全部回滚。对于大项目（1000+ 文件），一个文件失败导致全部工作浪费。可改为每 100 个文件提交一次子事务。 |
| 3 | **新发现** | `graph-handlers.ts:467-485` | **`handleDetectChanges` git diff 语义不一致**（前次未修复）：`git diff --name-only ${getDiffBranch}...HEAD` 失败时 fallback 到 `git diff --name-only HEAD`。结果语义从「分支差异」变为「未提交变更」。 |
| 4 | **新发现** | `sync.ts:160-162` | **`getIndexedCodebases()` 返回 identities 但日志打印了 identities**：`console.log` 打印 `indexedCodebases` 数组内容为 identities 而非 localPath，调试时可读性差。 |
