## 二、新发现问题

### P2 — 代码质量

| # | 类型 | 文件 | 问题 |
|---|------|------|------|
| 1 | **连锁反应** | `graph-store.ts:578-584` | **`regexToLike` 转义顺序错误，`%`/`_` 转义被后续 strip 撤销**：`replace(/%/g, '\\%')` 和 `replace(/_/g, '\\_')` 在 `replace(/[.*+?^${}()\|[\]\\]/g, '')` 之前执行。后者会匹配 `\%` 中的 `\` 并删除，恢复 `%` 为 LIKE 通配符。修复：将 `%/_` 转义移到最后，或调整 regex 不删除 `\%` 和 `\_` 中的 `\`。 |
| 2 | **新发现** | `handlers.ts:464-471` | **`forceReindex` 时 snapshot 一致性检查反写已清除的 snapshot**：`getIndexingCodebases` 命中时 `removeCodebaseCompletely` 清除了 snapshot，但后续 `snapshotHasIndex !== vectorDbHasIndex` 检查发现 VDB 有 index 而 snapshot 无，触发 recovery 逻辑写入 `completed` 状态。随后 `clearIndex` 清空 VDB，导致 snapshot 显示 completed 但 collection 为空。应在 `forceReindex` 路径跳过一致性检查。 |

### P3 — 边界情况

| # | 类型 | 文件 | 问题 |
|---|------|------|------|
| 3 | **新发现** | `milvus-restful-vectordb.ts:170-182` | **`makeRequest` 无重试逻辑**：HTTP 5xx 或网络错误直接抛出，无重试。Milvus 可能因内存不足、正在加载等原因临时不可用。应添加与 embedding 一致的指数退避重试。 |
| 4 | **新发现** | `graph-handlers.ts:769-774` | **`resolveCrossFileCalls` 中 `getNodeById` 循环调用**：`importEdges` 最多 200,000 条，每条 edge 有 sourceId 和 targetId，`getNodeById` 在循环中逐个调用。虽已去重（`nodeIds` Set），但仍为 N+1 查询。应添加 `getNodesByIdBatch(ids)` 批量查询。 |
| 5 | **新发现** | `graph-handlers.ts:467-485` | **`handleDetectChanges` git diff 语义不一致**：`git diff --name-only ${getDiffBranch}...HEAD` 失败时 fallback 到 `git diff --name-only HEAD`。前者显示「分支以来所有变更」，后者仅显示「未提交变更」。错误日志仅一行 `console.warn`，用户无感知语义差异。 |
