## 二、修复细节审查

### 2.1 Bug 2 修复：图索引后台化

```typescript
// handlers.ts:367 - 修复后
setImmediate(async () => {
    try {
        if (alreadyGraphIndexed && !args.force) {
            // 增量检测 + 索引
        } else {
            // 全量索引
        }
    } catch (e: any) {
        console.warn(`[INDEX] Graph indexing failed (non-fatal): ${e.message}`);
    }
});
```

**审查结论**: ✅ 正确。`setImmediate` 在当前事件循环的 I/O 回调之后、下次事件循环之前执行，确保 MCP 响应已发送。异常被 catch 包裹，不会导致进程崩溃。

### 2.2 Bug 3 修复：结构化变更检测

新方法 `detectChangedFiles()` 返回 `{ changedFiles: string[]; diffBranch: string } | null`，在 `handleIndex` 中直接使用结构化数据，不再解析文本。`handleDetectChanges`（MCP 工具，面向用户）仍然保留文本输出，两者互不干扰。

**审查结论**: ✅ 正确。代码清晰，职责分离。

### 2.3 Bug 4 修复：快照原子写入

```typescript
// snapshot.ts:745-748
const tmpPath = this.snapshotFilePath + '.tmp';
```

**审查结论**: ✅ 正确。`renameSync` 在同一文件系统上是原子操作，确保不会出现部分写入的文件。
